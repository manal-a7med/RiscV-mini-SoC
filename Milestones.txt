Why is dmem_rdata showing 'x' (Don't Care)?
In your simulation, dmem_rdata appears as red 'x' because of how the Verilog memory array is handled during a Write Cycle or an Idle Cycle:
    Uninitialized Memory: Since you are using $readmemh for the instruction memory but likely not for the data memory, the initial state of the data RAM is "undefined" (x).
    Write-Only Cycles: In your waveform, the CPU is currently performing writes to the stack or UART. During a write, the memory's read output typically holds its previous state or becomes undefined depending on your data_mem.v implementation.
  Simulation Artifact: Because you haven't read from an address that has been previously written to, the simulator has no "valid" data to drive onto the dmem_rdata bus, so it displays 'x'.
